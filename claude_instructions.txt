Local Contact Forms 2.0

I want to create a new Angular/netlify project.

The project will use Netlify functions and angular server side rendering.

This app will connect to google sheet api and pull tenant configs from the sheets.
Then the config will be loaded into each tenant, and the website user will be able to submit a contact form to that tenant via netlify email function.

Then each tenant will be configured to load when I visit a url like this:

https://localcontactforms.com/?id=ronnies-pizza-pace

Where ronnies-pizza-pace is the tenant.

There will be two netlify functions:

One to get the tenant config from the global google sheet
And one to submit the email on behalf of the tenant.

For server side rendering, each tenant will ahve their page title, meta keywords and meta description filled out per the tenant's config sheet.

There will be several main components:

contact form itself, which loads when going to https://localcontactforms.com/?id=ronnies-pizza-pace
A component to hold the ada statement
a component to hold the website terms of service
a component to hold the provacy policy

A loading component which will display while tenant config sheets are loading, and when submitting the email.

a component to hold the message when the form is submitted, whether is was successful or not, and will show the thank you message from the tenant config sheet:

post_submit_message	Thank you for contacting us!	Yes	Message shown to users after successful form submission.

Use angular services where proper.
